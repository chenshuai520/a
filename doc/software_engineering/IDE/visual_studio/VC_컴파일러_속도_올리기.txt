비졀씨 ++ 컴파일러가 기본적으로 하나의 프로젝트를 컴파일 하는데
 
멀티 쓰레드 처리를 안한다.
 
머 전체 빌드를 자주 할필요가 없다면 별로 활용도가 떨어지지만 리빌드 올을 자주 하는 사용자에게는..
 
치명적이다. CPU를 아무리 좋은것을 쓰나... 하드를 SSD를 쓰나...
 
어짜피 하나의 쓰레드로 하나의 프로젝트를 컴파일 하기때문에 차이를 느끼지 못한다...
 
여려 프로젝트를 한번에 전체 빌드를 할때는 따른 이야기겠지만... 그건 다음에 이야기 하기로 하고...
 
요즘 VC++개발자라면 보통 쿼드 코어에 하이퍼 쓰레딩을 지원 하는 CPU를 사용 할꺼라고 나는 감히...
 
단언한다. 그렇다면 컴파일 속도를 늘리기 위해서는 무었이 필요한가?? 좋은컴퓨터? 하지만 당신이 사용하는
 
컴퓨터 보다 2배이상 성은이 좋은것은 아마 당신의 연봉에 근접할껏이다. 그렇다면 툴의 성은을 좀더 올리는것이다.
 
그것은 바로 오늘 내가 이야기 하고자 하는 /MP옵션이다.
 
64비트 환경이라면 /MP256 쯤 줘보면 당장 리빌드 올의 컴파일 속도가 2배이상 빨라진것을 느낄것이다...
 
쓰레드를 256개를 활용하는거다. 그럼 CPU 사용율이 100%를 치면서 순식간에 컴파일 되는것을 느낄수 있을것이다.
 
32 비트 환경에서는 /MP32 이상은 무리다... 왜냐면 메모리가 4Gbyte 뿌니 지원을 안하는 환경에서
 
쓰레드를 256 개나 만들어봐야 메모리 페징잉때문에 CPU성능을 다 발휘 하지 못한다.
