[-] Building (Win32)
	[ref] ${QT_HOME}/README & ${QT_HOME}/qtbase/INSTALL
	[ref] http://doc.qt.io/qt-5/gettingstarted.html

	0. execute "Visual Studio Command Prompt"
	1. uncompress Qt source files to a directory
		e.g.) D:/Qt-5.5.1
		==> 위의 예와 같이 간단한 이름을 사용해야 함.
			그렇지 않을 경우 compile이 제대로 되지 않을 수 있음.
	2. set PATH
		${QT_HOME}/qtbase/bin
		${QT_HOME}/gnuwin32/bin
		${PERL_HOME}/bin
		${PYTHON_HOME}
	3. (optional) set the environment variable, QMAKESPEC
		set QMAKESPEC=win32-msvc2015
	4. change directory to ${QT_ROOT}
	    cd /D ${QT_ROOT}
	5. configure
		configure -prefix ${INSTALL_PREFIX} -debug-and-release -platform win32-msvc2015 -opensource -opengl desktop -nomake tests
		configure -prefix ${INSTALL_PREFIX} -debug-and-release -platform win32-msvc2015 -opensource -opengl desktop -no-openvg -I D:\usr\local\include -L D:\usr\local\lib
			INSTALL_PREFIX는 '\usr\local\Qt-5.5.1'과 같이 지정.
	    ==> to reconfigure:
	    	nmake confclean
	    	configure ...
	6. check Makefiles (==> not working)
		change win32-msvc10 to win32-msvc15 in Makefiles.
			${QT_HOME}/Makefile
			${QT_HOME}/qtbase/???/Makefile
	7. build
		-. nmake
		-. jom
			http://qt-project.org/wiki/jom
		-. mingw32-make
	8. install
		-. nmake install
		-. jom install
		-. mingw32-make install
		==> ${INSTALL_PREFIX}/include, lib, & bin, etc에 설치.

[-] Trouble Shooting
	-. Build 과정에서 error가 발생하면 source 전체를 지우고 다시 압축 해제 후 설치 진행. (?)
		[ref] http://stackoverflow.com/questions/34416611/error-when-building-qt-with-desktop-opengl
	-. manifest file 생성 error
		MSVC를 이용해 building하는 경우 mt.exe를 이용해 manifest file을 생성하는 과정에서
		manifest file을 생성하지 못하는 오류 발생.
		=> 다시 building하면 됨.
			원인을 알 수 없지만 Visual Studio IDE에서도 동일한 현상이 발생함.
