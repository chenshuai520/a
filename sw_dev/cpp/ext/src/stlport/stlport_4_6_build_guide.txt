<<  Building  >>

[-] STLport site로부터 library를 download
	-. http://www.stlport.org/
[-] 압축 해제
[-] STLport 환경 설정
	-. %STLPORT_ROOT%/stlport/stl_user_config.h 에서 원하는 항목을 적절히 설정
	-. enable _STLP_DEBUG
		#define _STLP_DEBUG 1
		==> Linking 항목 참조
[-] MSVC 환경변수 설정
	-. Visual Studio 6.0
		>< %VC6_ROOT%/Vc98/bin/vcvars32.bat 실행
		>< command
			vcvars32.bat > nul
	-. Visual Studio .NET 2003
		>< %VC71_ROOT%/Vc7/bin/vcvars32.bat 실행
		>< command
			vcvars32.bat > nul
[-] build STLport library
	-. Visual Studio 6.0
		>< initial building
			for non-unicode
				nmake -f vc6.mak
			for unicode
				nmake -f vc6-unicode.mak
		>< re-building
			for non-unicode
				nmake -f vc6.mak clean
				nmake -f vc6.mak all
			for unicode
				nmake -f vc6-unicode.mak clean
				nmake -f vc6-unicode.mak all
	-. Visual Studio .NET 2003
		>< initial building
			nmake -f vc71.mak
		>< re-building
			nmake -f vc71.mak clean
			nmake -f vc71.mak all
[-] install STLport library
	-. command
		>< Visual Studio 6.0
			for non-unicode
				nmake -f vc6.mak install
			for unicode
				nmake -f vc6-unicode.mak install
		>< Visual Studio .NET 2003
			nmake -f vc71.mak install
	-. resulting library files are installed into a subdirectory
		>< Visual Studio 6.0
			non-unicode & unicode에서 동일한 이름의 file 생성
			for non-unicode & unicode
				%VC6_ROOT%/Vc98/lib/stlport_vc6*.lib
				%WINDOWS_ROOT%/system32/stlport_vc6*.dll
		>< Visual Studio .NET 2003
			%VC71_ROOT%/Vc7/lib/stlport_vc71*.lib
			%WINDOWS_ROOT%/system32/stlport_vc71*.dll
	-. resulting include files are installed into a subdirectory
		>< Visual Studio 6.0
			%VC6_ROOT%/Vc98/include/stlport
		>< Visual Studio .NET 2003
			%VC71_ROOT%/Vc7/include/stlport
[-] set path
	-. Visual Studio IDE에서 아래의 directory를 include path의 '최상위'에 추가


<<  Linking  >>

[-] debug mode에서 STLport와 linking하는 경우 아래의 macro가 정의되어 있어야 함
	-. 해당 project의 compile시 아래의 define을 정의
		_STLP_DEBUG
	-. include 되는 "%STLPORT_ROOT%/stlport/stl_user_config.h"에서 아래와 같이 define을 활성화
		#define _STLP_DEBUG 1
		==> 파생문제점
			release building 시에도 _STLP_DEBUG macro가 define되어져 있게 됨
			#if defined(_DEBUG) || defined(DEBUG)
			#	define _STLP_DEBUG 1
			#endif
			

<<  Trouble Shooting  >>

[-] MSVC 7.1에서 native wchar_t를 지원하게 하기 위해서
	-. STLport compiling시 compile option을 변경하여야 함
		"%STLPORT_ROOT%/src/vc71.mak" file의 EXTRA_COMMON_FLAGS 항목에 아래의 option 추가
			/Zc:wchar_t
		MSVC 6.0에서는 사용할 수 없는 option임
		==> _NATIVE_WCHAR_T_DEFINED macro는 STLport 내부에서 define(?)해서 사용하는 내부 macro이므로
			재정의해서 사용할 수 없음
